AI Agent와 작업 환경 이해하기

SECTION 01

웹 기반 AI와 AI Agent, 무엇이 다른가

지금까지 사용한 도구들은 주로 웹 기반 AI였습니다. 대화창에 질문하고 답변을 받는 방식이며, 파일을
첨부하면 내용을 읽고 분석하거나 수정 방향을 제안받을 수도 있습니다.

하지만 실제 작업을 끝내는 흐름은 다릅니다. 웹 기반 AI는 사용자가 파일을 올리고, 답변을 받아서, 다시 사람이 직접 반영하고 실행해야 하는 경우가 많습니다. 반면 AI Agent는 지정한 작업 폴더를 기준으로 필요한 파일을 찾아보고, 수정하고, 실행 결과를 확인하면서 다음 작업으로 이어갈 수 있습니다.

웹 기반 AI
사용자가 이어서 처리하는 방식
  • 대화창에서 질문하고 답변을 받음
  • 필요한 파일은 사용자가 직접 업로드
  • AI는 초안·답변·수정안을 제시
  • 반영, 실행, 검증은 사용자가 이어서 수행
  • 개별 파일·문서 단위 작업에 적합
AI Agent
AI가 작업 흐름 전체를
직접 처리하는 방식
  • 지정한 작업 폴더를 기준으로 필요한
    파일을 확인
  • 파일 수정, 명령 실행, 결과 확인까지
    직접 진행
  • 오류가 나면 다음 수정으로 이어가며
    반복 수행
  • 중요한 작업은 승인 요청 후 진행
  • 프로젝트 구조와 변경 맥락을 함께 활용

핵심은 누가 다음 행동을 이어가느냐입니다. 웹 기반 AI는 답변을 주고, 그다음 행동은 사람이 이어갑니다. AI Agent는 목표를 주면 AI가 작업 흐름을 이어가고, 실제 영향이 큰 단계에서는 사용자에게 승인을 받아 진행합니다.

참고

AI Agent가 알아서 작업을 이어간다고 해서 완전히 제멋대로 움직인다는 뜻은 아닙니다. 파일 수정, 삭제, 명령어 실행처럼 실제 작업 환경에 영향을 주는 행동은 승인 요청이 표시될 수 있으며, 이때 사용자는 "AI가 지금 무엇을 하려는지" 확인하고 승인하면 됩니다.

SECTION 02

AI Agent란 무엇인가

AI Agent를 한 문장으로 표현하면 이렇습니다. 목표를 주면, 승인된 범위 안에서 계획을 세우고, 도구를 사용해 작업을 이어가는 AI입니다.

웹 기반 AI가 "이런 코드를 작성하면 됩니다"라고 알려준다면, 사용자는 그 코드를 복사해 파일에 반영하고 실행 결과를 직접 확인해야 합니다. 반면 AI Agent는 지정된 작업 폴더 안에서 파일을 수정하고, 실행 결과를 확인하며 다음 수정으로 이어갈 수 있습니다.

이것이 가능한 이유는 AI Agent가 도구(Tool)를 사용할 수 있기 때문입니다. 예를 들어 파일을 읽고 수정하거나, 터미널 명령어를 실행하고, 실행 결과를 확인하는 방식입니다. Antigravity는 이러한 작업을 AI Agent가 수행할 수 있도록 돕는 개발 환경입니다.

비유
비서 vs 대리인
웹 AI는 방향을 잡아주는 비서, AI Agent는 승인된 범위 안에서 작업을 이어가는 대리인
"이 보고서 수정해줘"라고 하면, 비서는 수정 방향을 알려주고 사용자가 직접 반영합니다. 대리인은 승인된 범위 안에서 파일을 열고 수정한 뒤 결과를 확인합니다. 결과는 비슷해 보여도, 작업을 어디까지 이어갈 수 있느냐가 다릅니다.

SECTION 04

승인 요청, 이런 뜻입니다

Antigravity에서 AI에게 작업을 시키다 보면 영어로 된 승인 팝업이 뜰 수 있습니다. 내용을 모른 채 무조건 Yes를 누르는 것은 위험할 수 있습니다. AI가 파일을 삭제하거나, 외부 패키지를 설치하거나, 터미널 명령어를 실행하려는 순간일 수 있기 때문입니다.

자주 뜨는 팝업의 의미는 다음과 같습니다.

GitHub, 왜 써야 하나요?

SECTION 01

GitHub은 왜 필요한가

AI Agent는 작업 폴더 안의 파일을 직접 확인하고 수정할 수 있습니다. 그래서 무엇이 바뀌었는지 확인하고, 필요하면 이전 상태로 되돌릴 수 있는 장치가 필요합니다.

GitHub은 작업 결과와 변경 이력을 남겨주는 온라인 저장소입니다. AI Agent가 만든 결과물을 안전하게 관리하기 위해 GitHub을 사용합니다.

구분 역할
Antigravity AI Agent와 함께 실제 작업을 진행하는 환경
작업 폴더 AI Agent가 파일을 열고 수정하는 공간
GitHub 작업 결과와 변경 이력을 관리하는 공간
핵심

GitHub은 AI Agent가 직접 일하는 공간이 아니라, 작업 결과와 변경 이력을 안전하게 관리하는 공간입니다.

SECTION 02

Repository와 Clone 이해하기

GitHub을 사용할 때 지금 단계에서 먼저 알아야 할 개념은 Repository와 Clone입니다.

Repository는 하나의 프로젝트를 담는 보관함입니다. 파일, 폴더 구조, 설명 문서, 기본 설정이 함께 들어 있습니다.

Clone은 GitHub에 있는 Repository를 내 PC 작업 폴더로 가져오는 과정입니다. Antigravity에서 Clone Repository를 선택하면, GitHub에 있는 프로젝트를 내 PC에 복사해 작업을 시작할 수 있습니다.

개념 쉽게 말하면 역할
Repository 프로젝트 보관함 프로젝트 파일과 구조를 담는 공간
Clone 가져오기 GitHub의 프로젝트를 내 PC로 가져오는 과정
작업 폴더 실제 작업 공간 AI Agent가 파일을 보고 수정하는 공간

즉, GitHub에 있는 Repository를 Clone해야 Antigravity에서 작업 폴더로 열 수 있습니다.

SECTION 03

사내 템플릿 Repository로 시작하는 이유

처음부터 빈 폴더에서 개발을 시작하면 폴더 구조, README* 작성 방식, 기본 설정을 모두 직접 정해야
합니다.

사내 템플릿 Repository는 이런 기본 틀을 미리 갖춰둔 양식입니다. 이 템플릿을 복사해서 시작하면, DSR 기준에 맞는 구조 안에서 프로젝트 내용을 채워 넣는 방식으로 작업할 수 있습니다.

또한 사내 템플릿에는 작업 안내 문서와 보안 관련 문서가 함께 포함되어 있습니다. AI Agent는 작업 폴더 안의 파일을 기준으로 작업하기 때문에, 템플릿을 Clone하면 AI Agent가 사내 기준과 보안 관련 안내를 참고할 수 있는 상태에서 작업을 시작할 수 있습니다.

회사 공문을 매번 빈 문서에서 새로 만들지 않고, 정해진 공문 양식을 복사해 작성하는 것과 같습니다.

* README : 프로젝트의 목적, 사용 방법, 파일 구조 등을 설명하는 안내 문서입니다.
* SECURITY.md : 프로젝트 작업 시 참고해야 할 보안 관련 안내 문서입니다.
